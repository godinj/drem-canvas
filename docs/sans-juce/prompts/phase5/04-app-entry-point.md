## Deliverables

### Migration

#### 1. src/Main.cpp

Replace the entire file. The current structure is:
- `DremCanvasApplication` class inheriting `juce::JUCEApplication` (+ `juce::Timer` on Linux)
- `initialise()` — creates platform window, GPU backend, renderer, AppController
- `shutdown()` — tears down in reverse

Replace with a plain `main()` function. The initialization and teardown logic
stays identical — only the JUCE scaffolding is removed.

**Linux implementation:**

```cpp
#include <filesystem>
#include "platform/linux/GlfwWindow.h"
#include "platform/linux/VulkanBackend.h"
#include "graphics/rendering/GpuBackend.h"
#include "graphics/rendering/Renderer.h"
#include "graphics/core/EventDispatch.h"
#include "ui/AppController.h"

int main (int /*argc*/, char* /*argv*/[])
{
    // Create GLFW window
    auto glfwWindow = std::make_unique<dc::platform::GlfwWindow> ("Drem Canvas", 1280, 800);

    // Set window icon
    auto exeDir = std::filesystem::canonical ("/proc/self/exe").parent_path();
    auto iconFile = exeDir / "drem-canvas.png";
    if (std::filesystem::exists (iconFile))
        glfwWindow->setWindowIcon (iconFile.string());

    // Create Skia Vulkan backend
    auto gpuBackend = std::make_unique<dc::platform::VulkanBackend> (
        glfwWindow->getHandle(),
        glfwWindow->getWidth(), glfwWindow->getHeight(),
        glfwWindow->getScale());

    auto renderer = std::make_unique<dc::gfx::Renderer> (*gpuBackend);

    // Create root widget
    auto appController = std::make_unique<dc::ui::AppController>();
    appController->setRenderer (renderer.get());

    float w = static_cast<float> (glfwWindow->getWidth());
    float h = static_cast<float> (glfwWindow->getHeight());
    appController->setBounds (dc::gfx::Rect (0, 0, w, h));

    auto eventDispatch = std::make_unique<dc::gfx::EventDispatch> (*appController);

    // Wire GLFW callbacks to EventDispatch
    glfwWindow->onMouseDown = [&] (const dc::gfx::MouseEvent& e)
    { eventDispatch->dispatchMouseDown (e); };
    glfwWindow->onMouseUp = [&] (const dc::gfx::MouseEvent& e)
    { eventDispatch->dispatchMouseUp (e); };
    glfwWindow->onMouseMove = [&] (const dc::gfx::MouseEvent& e)
    { eventDispatch->dispatchMouseMove (e); };
    glfwWindow->onMouseDrag = [&] (const dc::gfx::MouseEvent& e)
    { eventDispatch->dispatchMouseDrag (e); };
    glfwWindow->onKeyDown = [&] (const dc::gfx::KeyEvent& e)
    { eventDispatch->dispatchKeyDown (e); };
    glfwWindow->onKeyUp = [&] (const dc::gfx::KeyEvent& e)
    { eventDispatch->dispatchKeyUp (e); };
    glfwWindow->onWheel = [&] (const dc::gfx::WheelEvent& e)
    { eventDispatch->dispatchWheel (e); };

    glfwWindow->onResize = [&] (int newW, int newH)
    {
        auto* vulkan = static_cast<dc::platform::VulkanBackend*> (gpuBackend.get());
        vulkan->resize (newW, newH, glfwWindow->getScale());
        appController->setBounds (dc::gfx::Rect (0, 0,
            static_cast<float> (newW), static_cast<float> (newH)));
        renderer->forceNextFrame();
    };

    bool shouldQuit = false;
    glfwWindow->onClose = [&] () { shouldQuit = true; };

    appController->setNativeWindowHandle (glfwWindow->getHandle());
    appController->initialise();
    glfwWindow->show();

    // Main loop — replaces juce::Timer at 60Hz
    while (! shouldQuit && ! glfwWindow->shouldClose())
    {
        glfwWindow->pollEvents();
        appController->tick();
        renderer->renderFrame (*appController);
    }

    // Tear down in reverse order
    appController.reset();
    eventDispatch.reset();
    renderer.reset();
    gpuBackend.reset();
    glfwWindow.reset();

    return 0;
}
```

**macOS implementation:**

The macOS path is more complex because Metal rendering is driven by the
`MTKView` display link callback (`onFrame`). The approach:

```cpp
#include "platform/NativeWindow.h"
#include "platform/MetalView.h"
#include "platform/EventBridge.h"
#include "graphics/rendering/MetalBackend.h"
#include "graphics/rendering/GpuBackend.h"
#include "graphics/rendering/Renderer.h"
#include "graphics/core/EventDispatch.h"
#include "ui/AppController.h"

// Forward — implemented in a small .mm helper or inline
extern void dc_runNSApplication();

int main (int argc, char* argv[])
{
    // macOS requires NSApplication event loop. The actual setup happens
    // in applicationDidFinishLaunching (wired via dc_runNSApplication).
    dc_runNSApplication();
    return 0;
}
```

For macOS, there are two approaches — pick the one that fits the existing
platform layer best:

**Option A (preferred):** If `NativeWindow` already manages the NSApplication
run loop internally (check `NativeWindow.mm`), then the macOS path may just be:
```cpp
int main (int /*argc*/, char* /*argv*/[])
{
    // ... same setup as current initialise() ...
    nativeWindow->run();  // blocks until window close (NSApp run)
    // ... teardown ...
    return 0;
}
```

**Option B:** If the platform layer doesn't have a blocking run loop, create a
minimal `AppDelegate.mm` that wraps the setup/teardown in NSApplication lifecycle
callbacks. This is more invasive — only use if Option A isn't viable.

**Key points for both platforms:**
- The `appController->tick()` call must happen every frame (or at least 30Hz) for meter updates and message queue processing
- On macOS, call `tick()` inside the `onFrame` callback (already runs at display refresh rate)
- On Linux, call `tick()` inside the main while loop (runs at ~60Hz, limited by vsync or GLFW poll rate)
- Remove all `#include <JuceHeader.h>`
- Remove all `juce::` types
- Remove `START_JUCE_APPLICATION` macro
- Replace `JUCE_APPLICATION_NAME_STRING` and `JUCE_APPLICATION_VERSION_STRING` with `DC_APP_NAME` / `DC_APP_VERSION` from `dc/foundation/config.h` (if the window title or any code references these — check)

**Use `#if defined(__APPLE__)` / `#elif defined(__linux__)` preprocessor guards** to keep
both platforms in one file (matching the current structure), OR split into platform-specific
entry points if cleaner. Either approach is acceptable.
